# Bug Report Template

> Use this template to document bugs found during testing

# Promting 

The report should be based on the notes. Therefore, in the bug report, we anticipate the following components: Bug Summary, Bug Description, Test Data, Environment, Expected Output, Actual Output, and Steps to Reproduce. However, it is essential to adhere to the following constraints:

- Utilize ONLY the information provided in the notes

- Indicate any gaps as "[NEEDS CLARIFICATION]"

- Refrain from fabricating details

- Request any missing information
